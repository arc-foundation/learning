Git Basics: A Quick Guide

Introduction

Git is a distributed version control system that helps you track changes in your code, collaborate with others, and manage your project's history. This guide covers the basic Git commands and concepts.

Geting Started

  1. Installing Git

    2. Before using Git, make sure it's installed on your machine. You can download it from here

  2. Configuring Git

    3. After installation, configure your name and email:

    git config --global user.name "Your Name"
    git config --global user.email "your.email@example.com"

    Creating a New Repository

  3. Initializing a Repository

    4. git init

  4. Adding Files

    5. git add <filename>
    git add .

  5. Committing Changes

    6. git commit -m <commit message>

  6. Push the Change to server

    7. git push origin <branch name>

    Working with Branches

    1. Creating a Branch

      2. Create a new branch for a feature or bug fix:

      git branch <feature-branch>

    2. Switching Branches

      3. Switch to the new branch:

      git checkout feature-branch

    3. Merging Branches

      4. git checkout main
      git merge feature-branch

      Collaboration

      1. Cloning a Repository

        2. Clone a remote repository to your local machine:

        git clone https://github.com/username/repo.git

      2. Pulling Changes

        3. Update your local repository with remote changes:

        git pull origin main

      3. Pushing Changes

        4. Push your local changes to the remote repository:

        git push origin main

        Additional Commands

        • git status:Check the status of your repository.
        • git log:View commit history.
        • git diff:Show changes between commits.
